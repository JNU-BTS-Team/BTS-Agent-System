# 4.2 视图保护 - 脱敏视图方案

## 方案一：用户信息脱敏视图（view_user_masked）

| 项目 | 内容 |
|------|------|
| **原始表（含敏感字段）** | **users 表** |
| **视图名称及定义** | **view_user_masked**<br/>从 users 表中查询用户信息，但对敏感字段进行脱敏处理（隐藏密码、脱敏姓名和电话） |
| **隐藏的敏感字段** | **password（密码哈希值）、real_name（真实姓名）、phone（联系电话）** |
| **授权给的用户** | **doctor 和 viewer** |
| **安全意义说明** | 防止用户账户密码泄露和个人隐私信息暴露。原始 users 表包含完整的密码哈希值（256 位）、真实姓名和联系电话等高度敏感的系统安全信息。通过视图完全隐藏 password 字段（不在视图中显示），防止恶意用户或未授权人员获取密码哈希值进行破解攻击。同时对 real_name 脱敏为首字+**（如：王**）、对 phone 脱敏为 XXX****XXXX 格式（如：139****7919），doctor 医生可以通过脱敏的联系方式进行工作协调，viewer 审计员可以核查用户账户信息，但双方都无法获取完整的隐私信息和系统安全信息，有效防止社工攻击、账户劫持和数据泄露风险 |

### SQL 代码：

```sql
-- 创建用户信息脱敏视图
DROP VIEW IF EXISTS view_user_masked;
CREATE VIEW view_user_masked AS
SELECT
    id AS 用户ID,
    username AS 账号,
    -- 真实姓名脱敏：显示首字 + **
    CONCAT(
        SUBSTR(real_name, 1, 1),
        REPEAT('*', CHAR_LENGTH(real_name) - 1)
    ) AS 真实姓名,
    role AS 角色,
    CASE role
        WHEN 'admin'  THEN '系统管理员'
        WHEN 'doctor' THEN '医生'
        WHEN 'viewer' THEN '审计员'
    END AS 角色说明,
    department AS 所属科室,
    title AS 职位,
    -- 电话号码脱敏：显示前3位+****+后4位 (如：139****7919)
    CONCAT(
        SUBSTR(phone, 1, 3),
        '****',
        SUBSTR(phone, -4)
    ) AS 联系电话,
    created_at AS 注册时间
FROM users
WHERE real_name IS NOT NULL;

-- 授权给 doctor 和 viewer
GRANT SELECT ON SECD.view_user_masked TO 'doctor'@'localhost';
GRANT SELECT ON SECD.view_user_masked TO 'viewer'@'localhost';
FLUSH PRIVILEGES;
```

### 脱敏效果示例：

| 字段 | 原始值 | 脱敏后 | 说明 |
|------|--------|--------|------|
| real_name | 王雅宁 | 王** | 只显示首字 + ** |
| phone | 13734567890 | 137****7890 | 显示前3位 + **** + 后4位 |
| password | a665a45920422f9... | （不显示） | 完全隐藏在视图中 |

## 方案二：患者信息脱敏视图（view_patient_masked）

| 项目 | 内容 |
|------|------|
| **原始表（含敏感字段）** | **patients 表** |
| **视图名称及定义** | **view_patient_masked**<br/>从 patients 表中查询患者信息，但对个人隐私敏感字段进行脱敏处理（脱敏出生日期、电话、住址） |
| **隐藏的敏感字段** | **birthday（出生日期）、phone（联系电话）、address（住址）** |
| **授权给的用户** | **doctor 和 viewer** |
| **安全意义说明** | 保护患者隐私信息，符合医疗数据保护法规（HIPAA、GDPR）。原始 patients 表包含患者完整的出生日期、联系电话和详细住址等高度敏感的个人隐私信息，这些信息一旦泄露可能导致身份盗用、诈骗、骚扰等严重后果。通过视图对敏感字段进行脱敏处理：出生日期只显示年份（如：1967-**）隐藏月日便于身份推断、电话脱敏为 139****7919 格式隐藏完整号码、地址仅显示前5个字符（如：北京市朝阳**）隐藏完整住址。doctor 医生在进行患者诊疗和随访工作时可以通过脱敏信息进行核实，viewer 审计员可以进行数据审计工作，但双方都无法获取足以进行身份盗用或骚扰的完整隐私信息。即使 web 界面实际不展示这些字段，视图层仍提供了第二道防线，确保即使数据被意外访问也是脱敏状态，最大化保护患者隐私安全 |

### SQL 代码：

```sql
-- 创建患者信息脱敏视图
DROP VIEW IF EXISTS view_patient_masked;
CREATE VIEW view_patient_masked AS
SELECT
    id AS 患者ID,
    patient_id AS 患者编号,
    name AS 患者姓名,
    gender AS 性别,
    age AS 年龄,
    -- 出生日期脱敏：只显示年份，隐藏月日
    CONCAT(YEAR(birthday), '-**') AS 出生日期,
    -- 电话号码脱敏：显示前3位+****+后4位 (如：139****7919)
    CONCAT(
        SUBSTR(phone, 1, 3),
        '****',
        SUBSTR(phone, -4)
    ) AS 联系电话,
    -- 住址脱敏：只保留前5个字符，后续用**表示
    CONCAT(
        SUBSTR(address, 1, 5),
        '**'
    ) AS 住址,
    created_at AS 注册时间
FROM patients
WHERE phone IS NOT NULL;

-- 授权给 doctor 和 viewer
GRANT SELECT ON SECD.view_patient_masked TO 'doctor'@'localhost';
GRANT SELECT ON SECD.view_patient_masked TO 'viewer'@'localhost';
FLUSH PRIVILEGES;
```

### 脱敏效果示例：

| 字段 | 原始值 | 脱敏后 | 说明 |
|------|--------|--------|------|
| birthday | 1967-05-20 | 1967-** | 只显示年份，隐藏月日 |
| phone | 13800138001 | 138****8001 | 显示前3位 + **** + 后4位 |
| address | 北京市朝阳区建国路1号 | 北京市朝** | 只显示前5个字符 |

## 脱敏函数详解

### 1. 姓名脱敏函数
```sql
CONCAT(
    SUBSTR(real_name, 1, 1),                    -- 取首字
    REPEAT('*', CHAR_LENGTH(real_name) - 1)    -- 后续用*填充
) AS 真实姓名
```

**原理**：
- `SUBSTR(real_name, 1, 1)` - 取姓名的第一个字符
- `CHAR_LENGTH(real_name) - 1` - 计算剩余字符数
- `REPEAT('*', N)` - 用 N 个 * 替代
- `CONCAT()` - 合并字符

**示例**：
- 王雅宁 (3字) → 王 + ** = 王**
- 李 (1字) → 李 + (0个*) = 李
- 张三丰 (3字) → 张 + ** = 张**

---

### 2. 电话号码脱敏函数
```sql
CONCAT(
    SUBSTR(phone, 1, 3),      -- 前3位
    '****',                    -- 中间4个*
    SUBSTR(phone, -4)          -- 最后4位
) AS 联系电话
```

**原理**：
- `SUBSTR(phone, 1, 3)` - 取前3位
- `SUBSTR(phone, -4)` - 取最后4位（-4 表示倒数第4位开始）
- `CONCAT()` - 合并为 XXX****XXXX 格式

**示例**：
- 13734567890 → 137 + **** + 7890 = 137****7890
- 13800138001 → 138 + **** + 8001 = 138****8001

---

### 3. 出生日期脱敏函数
```sql
CONCAT(YEAR(birthday), '-**') AS 出生日期
```

**原理**：
- `YEAR(birthday)` - 提取出生年份
- 后续用 -** 表示隐藏的月日
- 保留年份用于统计和年龄验证

**示例**：
- 1967-05-20 → 1967-**
- 1990-12-25 → 1990-**

---

### 4. 地址脱敏函数
```sql
CONCAT(
    SUBSTR(address, 1, 5),    -- 前5个字符
    '**'                       -- 后续用**表示
) AS 住址
```

**原理**：
- `SUBSTR(address, 1, 5)` - 取前5个字符（通常包含省市）
- 后续用 ** 隐藏具体街道和门牌

**示例**：
- 北京市朝阳区建国路1号 → 北京市朝** （前5字）
- 上海市浦东新区世纪大道100号 → 上海市浦东** （前5字）

## 应用场景

### doctor 医生通过脱敏视图查询用户
- **用途**：工作协调、团队沟通
- **可看到**：其他医生的脱敏姓名（首字+**）、脱敏电话（XXX****XXXX）
- **看不到**：完整密码哈希值、完整个人信息

### viewer 审计员通过脱敏视图审计数据
- **用途**：系统审计、权限检查、合规验证
- **可看到**：所有用户/患者的脱敏基本信息
- **看不到**：密码、完整隐私数据

### 患者诊疗场景
- **doctor 查询**：通过脱敏视图查询患者基本信息（年龄、性别、诊断日期）
- **隐私保护**：无法看到完整出生日期、电话号码、住址等隐私信息
- **法规符合**：符合 HIPAA（美国医疗隐私法）、GDPR（欧盟通用数据保护条例）等规范
